Neo-bank fraud ops prototype

LedgerGuard is a training-grade fraud detection system with an internal Ops workflow. Ops users upload transactions, score fraud risk, triage flagged activity, and export flagged subsets for review.

Open Ops dashboard Ops sign in View on GitHub
Ops access requires an authorised staff account. Create one locally using Django admin or createsuperuser.

What you can demo

  • Upload CSV and score transactions with the current baseline
  • Review and filter a session-backed scored run without rescoring
  • Export flagged transactions as a clean CSV for Ops review
  • Enforce Ops access controls and predictable login redirects

Internal Ops surface

Routes live under the Ops namespace and require authentication. Staff-only access protects the surface from accidental exposure.

Session-backed run contract

The dashboard stores the last scored run in session, then applies filters on the server. Export reads from the same contract to keep behaviour consistent.

Test-gated behaviour

Request flows, exports, and error states have tests that lock in product behaviour. Changes surface quickly in CI.
